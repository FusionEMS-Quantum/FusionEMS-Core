# DSAR Procedure

| Field | Value |
|---|---|
| Policy ID | DSAR-001 |
| Version | 1.0 |
| Effective Date | March 9, 2026 |
| Owner | Privacy Officer |

## Intake Channels

- Web form
- Email (privacy inbox)
- Phone support escalation

## Identity Verification

Requester identity is verified before release or modification actions (multi-factor checks and contextual validation).

## Request Types

- Access/copy
- Correction/amendment
- Deletion (where legally permitted)
- Restriction/objection
- Portability (where applicable)

## Timelines

- Baseline: 30 days
- Extension: additional 30 days with notice and rationale
- HIPAA right-of-access handling follows HIPAA timelines and constraints

## Fulfillment Workflow

1. Log request and assign case ID
2. Verify identity and request scope
3. Query data stores and logs (RDS/S3/audit sources)
4. Prepare response package
5. QA/legal review for exemptions
6. Deliver securely and close with evidence

## Denial Grounds

Legal hold, safety exceptions, third-party confidentiality, or other lawful exemptions must be documented with rationale.

## Evidence

- DSAR register
- Fulfillment artifacts
- Decision and communication log
